Before we start
What exactly IS a RAG? 🤔
The Plain-English Explanation

Imagine you have a very smart friend (an AI) who knows everything — but hasn't seen your Superstore data. A RAG is how you hand that friend your data first, so when you ask "Which region had the worst profit in Q3?", they answer based on your actual numbers — not just general knowledge.

RAG stands for Retrieval-Augmented Generation: you retrieve relevant chunks of your data, then augment (add) them to the AI's prompt, so it generates a grounded, accurate answer.

Process the Superstore Data

▼
What you'll learn

What embeddings are and how ChromaDB stores them locally on your SSD. This is the step that makes your data searchable by the AI. The embedding model runs locally (small, fast, no RAM issues) — only the final AI answer goes through Gemini.

What is an Embedding? Simply explained

An embedding converts text into a list of numbers. "High profit in West region" and "West region is profitable" produce very similar number lists — so the system knows they mean the same thing and groups them together. ChromaDB stores all these numbers and instantly finds the most relevant rows for any question you ask.

1

Export Superstore data from Tableau

In Tableau Desktop, open your Superstore workbook → go to the Data Source tab → right-click the Orders sheet → Export Data to CSV → save as superstore.csv in C:\Projects\tableau-rag\.

Alternative: Tableau installs the Superstore file at C:\Program Files\Tableau\Tableau [version]\Datasources\. Open in Excel, go to Orders sheet, Save As CSV.
2

Check your column names first!

Open superstore.csv in VS Code and look at the header row before writing any code. Common variations:

# Typical Superstore column names — verify yours match: Order ID, Order Date, Ship Mode, Customer Name, Segment Region, State, Category, Sub-Category, Product Name Sales, Quantity, Discount, Profit # Note: might be "Sub Category" (no hyphen) — check carefully!
3

Create ingest.py — build the knowledge base

Run this once only. It saves ChromaDB to your SSD and you never need to run it again unless your data changes.

# ingest.py — Run ONCE to build your ChromaDB knowledge base import pandas as pd from langchain_community.vectorstores import Chroma from langchain_community.embeddings import HuggingFaceEmbeddings # Load Superstore CSV df = pd.read_csv("superstore.csv") print(f"Loaded {len(df)} rows") print("Columns:", df.columns.tolist()) # Check names here! # Convert each row into a descriptive sentence def row_to_text(row): return ( f"Order {row['Order ID']}: {row['Category']} › {row['Sub-Category']} — " f"{row['Product Name']}. Region: {row['Region']}, State: {row['State']}. " f"Sales: ${row['Sales']:.2f}, Profit: ${row['Profit']:.2f}, " f"Quantity: {row['Quantity']}, Discount: {row['Discount']:.0%}. " f"Segment: {row['Segment']}, Ship Mode: {row['Ship Mode']}." ) texts = df.apply(row_to_text, axis=1).tolist() print(f"Sample: {texts[0]}") # Preview first converted row # Embedding model — runs locally, tiny RAM footprint (~200MB) embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2") # Build and save ChromaDB to your SSD print("Building ChromaDB... (2-5 mins)") vectorstore = Chroma.from_texts( texts=texts, embedding=embeddings, persist_directory="./chroma_db" ) print(f"✅ Done! {len(texts)} rows stored in ChromaDB.") print("chroma_db/ folder saved — never run this again!")
⚠️ KeyError? Means a column name in the code doesn't match your CSV. Look at the printed column list and fix the names in row_to_text. The most common mismatch is Sub-Category vs Sub Category.

Build the RAG Backend
~45 min · LangChain + Gemini + FastAPI
▼
What you'll learn

How a LangChain RAG chain works: it searches ChromaDB for the most relevant Superstore rows, builds a prompt, and sends it to Gemini — which replies in 2–3 seconds. Then you'll wrap it in FastAPI so Tableau can talk to it via HTTP.

1

Create rag_chain.py — the core RAG logic

# rag_chain.py from langchain_community.vectorstores import Chroma from langchain_community.embeddings import HuggingFaceEmbeddings from langchain_google_genai import ChatGoogleGenerativeAI from langchain.chains import RetrievalQA from langchain.prompts import PromptTemplate # Must match the model used in ingest.py! embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2") # Load ChromaDB from your SSD vectorstore = Chroma( persist_directory="./chroma_db", embedding_function=embeddings ) # Connect to Gemini — paste your API key here llm = ChatGoogleGenerativeAI( model="gemini-1.5-flash", google_api_key="YOUR_API_KEY_HERE", temperature=0.1 # Low = more factual, less creative ) # Prompt tells Gemini its role and how to use the data prompt_template = """You are a business data analyst for Tableau Superstore data. Use only the data context below to answer the question. Be concise and specific. If the answer is not in the context, say "I don't have enough data to answer that." Context (relevant Superstore records): {context} Question: {question} Answer:""" prompt = PromptTemplate( template=prompt_template, input_variables=["context", "question"] ) # k=8 retrieves 8 most relevant rows — good balance for Superstore qa_chain = RetrievalQA.from_chain_type( llm=llm, chain_type="stuff", retriever=vectorstore.as_retriever(search_kwargs={"k": 8}), chain_type_kwargs={"prompt": prompt}, return_source_documents=True ) def ask(question: str) -> dict: result = qa_chain.invoke({"query": question}) return { "answer": result["result"], "sources_count": len(result["source_documents"]) }
2

# test_rag.py from rag_chain import ask print("Testing RAG with Gemini...") result = ask("Which product category has the highest total profit?") print("Answer:", result["answer"]) print("Based on:", result["sources_count"], "records")
python test_rag.py # Should answer in 2-3 seconds — Gemini is fast! ✅

# api_server.py from fastapi import FastAPI from fastapi.middleware.cors import CORSMiddleware from pydantic import BaseModel from rag_chain import ask app = FastAPI(title="Superstore RAG API") app.add_middleware( CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"] ) class Question(BaseModel): question: str @app.post("/ask") async def answer_question(q: Question): return ask(q.question) @app.get("/") async def health(): return {"status": "Superstore RAG running! ✦ Powered by Gemini"}
# Start the server: uvicorn api_server:app --reload --port 8000 # Visit in browser: http://localhost:8000 # Should show: {"status": "Superstore RAG running! ✦ Powered by Gemini"}

Dashboard Context Injection
~45 min · Make Gemini "see" your active filters & selections
▼
What you'll learn

The Tableau Extensions API (free, works on free Desktop ✅) lets your HTML page read live dashboard state — active filters, selected marks, parameter values. You'll inject this context silently into every Gemini question so the AI answers about exactly what the user is looking at.

Simply explained

You filter to "West Region, Furniture". You type "how are profits?". Without Phase 6: Gemini answers about all regions and all categories. With Phase 6: Gemini reads your filters automatically and answers about West Region + Furniture only — without you typing that in the chat.

1

Add Tableau Extensions API to your HTML

Add this inside <head> in extension/index.html:

<!-- Tableau's official JS library — free CDN link --> <script src="https://cdn.jsdelivr.net/npm/tableau.extensions.api/lib/tableau.extensions.1.latest.min.js"></script>
2

Create extension/context.js

// context.js — reads live Tableau dashboard state async function initTableau() { return new Promise((resolve, reject) => { tableau.extensions.initializeAsync().then(resolve).catch(reject); }); } async function getActiveFilters() { const dashboard = tableau.extensions.dashboardContent.dashboard; const filterSummary = []; for (const ws of dashboard.worksheets) { try { const filters = await ws.getFiltersAsync(); for (const f of filters) { if (f.filterType === 'categorical') { const vals = f.appliedValues.map(v => v.formattedValue); if (vals.length > 0 && vals.length < 20) filterSummary.push(`${f.fieldName}: ${vals.join(', ')}`); } else if (f.filterType === 'range') { filterSummary.push(`${f.fieldName}: ${f.minValue?.formattedValue} to ${f.maxValue?.formattedValue}`); } } } catch(e) {} } return [...new Set(filterSummary)]; } async function getSelectedMarks() { const dashboard = tableau.extensions.dashboardContent.dashboard; const markSummary = []; for (const ws of dashboard.worksheets) { try { const marks = await ws.getSelectedMarksAsync(); if (marks.data.length > 0) { const cols = marks.data[0].columns.map(c => c.fieldName); marks.data[0].data.slice(0, 5).forEach(row => { markSummary.push(cols.map((c,i) => `${c}=${row[i].formattedValue}`).join(', ')); }); } } catch(e) {} } return markSummary; } async function buildDashboardContext() { const parts = []; const filters = await getActiveFilters(); if (filters.length > 0) parts.push(`Active filters: ${filters.join(' | ')}`); const marks = await getSelectedMarks(); if (marks.length > 0) parts.push(`Selected: ${marks.join(' ; ')}`); return parts.length > 0 ? `[Dashboard Context: ${parts.join('. ')}]\n\n` : ''; }
